Shipment

AWB shipment tracking → Amazon shipment-update files. One AWB = one shipped batch.

This workflow is being set up. The steps below are the agreed flow — we'll wire each one up (and you'll share the Amazon file template).
Workflow
1
Collect shipment emails. The Apps Script gathers Excel attachments from the SHIPMENT UPDATES label into Drive; the system saves each AWB's details.
2
Upload the unshipped-orders report (txt). Any order not in it counts as cancelled.
3
Track every AWB against the status sheet — delivered or not, plus the last ship date. An Apps Script pings the system the moment an AWB is delivered.
4
On delivery, the system prompts you to refresh the unshipped report, then generates the Amazon shipment-update file for that AWB's order IDs (minus cancelled) and emails it to you.
5
Dashboard sheet keeps the running data for the project.
